IAM Role - AWS

IAM Role

IAM identity that you can create in your account that has specific permissions. An IAM role has some similarities to an IAM user. Roles and users are both AWS identities with permissions policies that determine what the identity can and cannot do in AWS. However, instead of being uniquely associated with one person, a role is intended to be assumable by anyone who needs it. Also, a role does not have standard long-term credentials such as a password or access keys associated with it. Instead, when you assume a role, it provides you with temporary security credentials for your role session.

Roles can be used by the following:

  • An IAM user in the same AWS account as the role
  • An IAM user in a different AWS account than the role
  • A web service offered by AWS such as Amazon Elastic Compute Cloud (Amazon EC2)
  • An external user authenticated by an external identity provider (IdP) service that is compatible with SAML 2.0 or OpenID Connect, or a custom-built identity broker.

Creating IAM Role

  1. Sign in to the AWS Management Console:
    2. Open your web browser and navigate to the AWS Management Console. Sign in with your AWS account credentials.
  2. Navigate to IAM:
    3. In the AWS Management Console, find the "Services" dropdown, and under "Security, Identity, & Compliance," select "IAM."
  3. Access the Roles Page:
    4. In the IAM dashboard, click on "Roles" in the left navigation pane to access the Roles page.
  4. Create a New Role:
    5. Click the "Create role" button to start the process of creating a new IAM role.
  5. Choose the Trusted Entity Type:
    6. Select the trusted entity type that will assume the role. For example, choose "AWS service" if you want an AWS service to assume the role.
  6. Select Use Case:
    7. Choose a use case or service that will assume the role. The available options depend on the trusted entity type selected. For example, for an EC2 instance, choose "EC2."
  7. Attach Permissions Policies:
    8. Attach policies to the role to define the permissions it will have. You can choose from existing policies or create custom policies. This step determines what actions the role is allowed to perform.
  8. Set Role Name and Review:
    9. Enter a name for the role and optionally provide a description. Review the configuration to ensure it meets your requirements.
  9. Add Tags (Optional):
    10. You can add tags to the role for better organization and management. Tags are key-value pairs.
  10. Review and Create:
    11. Review the role details, permissions, and settings. If everything looks correct, click on the "Create role" button.
  11. Role Created:
    12. Once the role is created, you'll see a confirmation screen with details about the new role. Take note of the Role ARN (Amazon Resource Name).
